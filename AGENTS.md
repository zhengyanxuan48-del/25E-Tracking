# AGENTS.md

## 项目背景

本项目是基于 **天猛星 MSPM0G3507** 的小车循迹控制代码。硬件使用 **八路红外循迹模块**，任务目标是在 **100cm × 100cm 的正方形黑线边框** 上实现 **快且稳** 的闭环循迹。

核心优先级：

1. 稳定不丢线。
2. 转角识别可靠，尤其是正方形四个直角。
3. 在稳定前提下尽量提高速度。
4. 保持代码改动小、可验证、易回退。

## 循迹任务相关规则

### 八路红外数据处理

处理八路红外循迹时，应优先确认以下问题：

1. 黑线检测逻辑是高电平有效还是低电平有效。
2. 八路传感器顺序是否从左到右一致。
3. 是否存在传感器阈值或二值化逻辑。
4. 是否处理了全白、全黑、单边丢线、十字或直角状态。
5. 是否记录上一次有效偏差用于丢线恢复。
6. 是否有滤波或防抖，避免高速时抖动。

推荐权重示例：

```c
static const int8_t line_weight[8] = {-7, -5, -3, -1, 1, 3, 5, 7}; // 八路红外权重：左负右正，用于计算循迹偏差
```

如需新增权重或阈值，必须带中文注释。

### 正方形黑线边框特性

目标场地为 **100cm × 100cm 正方形黑线边框**，算法需要重点处理：

1. 长直线高速稳定。
2. 四个 90° 直角弯。
3. 进入直角前的减速或转向增强。
4. 弯中短时间全黑、全白或单侧多传感器触发。
5. 出弯后快速回正。
6. 避免在角点来回振荡。

### 控制策略建议

在没有用户明确要求重构前，优先采用最小增强：

1. 普通直线：使用 PD 或 PID 控制。
2. 小偏差：保持较高基础速度。
3. 大偏差：降低基础速度并增大差速。
4. 检测到直角：短时间进入转角状态。
5. 丢线：根据上一次有效偏差方向恢复。
6. 出弯：重新检测到中间传感器后恢复普通循迹。

新增参数示例必须写中文注释：

```c
static int16_t debug_base_speed = 520; // 调试参数：基础速度，数值越大直线越快
static int16_t debug_turn_speed = 360; // 调试参数：直角转弯速度，用于提高过弯稳定性
static float debug_kp = 22.0f; // 调试参数：比例系数，用于控制偏差响应强度
static float debug_kd = 8.0f; // 调试参数：微分系数，用于抑制高速循迹抖动
static uint16_t debug_lost_timeout_ms = 80; // 调试参数：丢线保护时间，用于判断是否进入找线状态
```

## MSPM0G3507 平台注意事项

修改 MSPM0G3507 相关代码时必须注意：

1. 不随意改动启动文件、链接脚本、SysConfig 自动生成文件。
2. 不随意修改时钟配置，除非任务明确要求。
3. 不随意修改中断优先级，除非问题明确来自中断。
4. 不使用 STM32 专用 HAL API。
5. 不引入与 MSPM0G3507 不兼容的寄存器或库。
6. 修改 GPIO、PWM、定时器、ADC、UART 前必须先确认现有初始化代码。
7. 若代码来自 TI DriverLib 或 SysConfig，优先遵守现有工程风格。

## 本项目优化方向

后续优化循迹代码时，优先围绕以下方向进行：

1. 八路红外权重计算是否准确。
2. 黑白阈值是否可靠。
3. 全白丢线恢复是否稳定。
4. 直角弯识别是否及时。
5. 直角弯是否有保持时间或退出条件。
6. PWM 输出是否限幅。
7. 左右电机速度是否存在静差补偿。
8. 高速直线是否抖动。
9. 出弯后是否快速回正。
10. 是否能稳定完成 100cm × 100cm 正方形黑线边框循迹。

## 最终原则

本项目的第一目标不是写最多代码，而是用最小、可验证、可回退的修改，让 MSPM0G3507 小车在八路红外循迹条件下更快、更稳地完成 100cm × 100cm 正方形黑线边框循迹任务。
